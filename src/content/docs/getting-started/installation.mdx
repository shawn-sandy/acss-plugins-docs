Plugins are distributed through the Claude Code plugin marketplace under the
`shawn-sandy` namespace.

## Install acss-kit

acss-kit ships components, themes, and utility-class commands (`/utility-add`,
`/utility-list`, `/utility-tune`, `/utility-bridge`) all in one plugin.

<Steps>

1. **Add the marketplace** to your Claude Code session:

   ```text
   /plugin marketplace add shawn-sandy/agentic-acss-plugins
   ```

2. **Install acss-kit** (components + themes + utility classes):

   ```text
   /plugin install acss-kit@shawn-sandy-agentic-acss-plugins
   ```

3. **Bootstrap your project** — run `/setup` once to initialize the foundation:

   ```text
   /setup
   ```

4. **Verify the install** — list available components to confirm acss-kit is
   active:

   ```text
   /kit-list
   ```

   You should see the full component catalogue grouped by category.

</Steps>

<Aside type="tip">
  `/utility-add`, `/utility-list`, `/utility-tune`, and `/utility-bridge` are
  all part of acss-kit since v1.0.0. No separate utility plugin install is
  needed.
</Aside>

## Also install style-agent (optional)

style-agent adds framework-agnostic CSS authoring commands (`/css-to-class`,
`/inline-style-to-class`, `/create-utilities`). Install it alongside acss-kit
if you want those workflows:

```text
/plugin install style-agent@shawn-sandy-agentic-acss-plugins
```

## Verify installation

After installing, confirm the slash commands are available:

```text
/kit-list          # lists all available components
/utility-list      # lists all utility families and classes
```

Both commands are read-only and make no changes to your project — they're safe
to run at any time to explore what's available.

## Updating the plugins

To get the latest version of a plugin:

```text
/plugin update acss-kit@shawn-sandy-agentic-acss-plugins
/plugin update style-agent@shawn-sandy-agentic-acss-plugins
```

## Local development install

If you've cloned the plugin repo and want to test local changes without
publishing:

```bash
# From the acss-plugins repo root
claude --plugin-dir ./plugins/acss-kit
```

This loads acss-kit directly from the local directory, bypassing the
marketplace.
