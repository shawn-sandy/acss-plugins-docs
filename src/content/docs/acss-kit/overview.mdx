**acss-kit** is a Claude Code plugin that generates accessible React components and CSS themes for [fpkit/acss](https://github.com/shawn-sandy/acss) projects.

## What it generates

<CardGrid>
  <Card title="Components" icon="puzzle">
    Self-contained `.tsx` + `.scss` file pairs written to
    `src/components/fpkit/`. No runtime npm dependency â€” you own the source.
  </Card>
  <Card title="Themes" icon="seti:css">
    OKLCH-derived CSS custom property files in `src/styles/theme/`. 18 semantic
    color roles, WCAG AA validated, light + dark.
  </Card>
</CardGrid>

## Skills

| Skill         | Triggered by                                                                    | Purpose                                                                 |
| ------------- | ------------------------------------------------------------------------------- | ----------------------------------------------------------------------- |
| `setup`       | `/setup`                                                                        | One-shot project bootstrap (ui.tsx, target, optional seed theme)        |
| `components`  | `/kit-add`, `/kit-create`, natural language ("create a button", "build a form") | Full component generation, Creator Mode, Form Mode, and HTML Target     |
| `styles`      | All `/theme-*` and `/color-scale` commands, natural language ("warmer button")  | OKLCH palette math, WCAG validation, role catalogue, style-tune routing |
| `style-tune`  | `/style-tune` or natural language ("warmer button", "softer card")              | Dispatch router â€” resolves intent and delegates to styles or components |
| `kit-sync`    | `/kit-sync`, `/kit-update`                                                      | Bulk install and safe-update with SHA256 manifest drift detection       |
| `prompt-book` | `/prompt-book`                                                                  | Displays copy-paste prompts for all slash commands                      |
| `utilities`   | `/utility-add`, `/utility-list`, `/utility-tune`, `/utility-bridge`             | Atomic-CSS utility bundle management, token bridge, responsive variants |

## Key design decisions

**Vendored source.** Components are written as source files into your project. There's no runtime `@fpkit/acss` import â€” your bundle doesn't grow, and you can edit the generated files freely.

**Generation contract.** Before writing any file, `/kit-add` shows a preview listing exactly which files will be created or skipped. You always know what's about to happen.

**Dependency resolution.** If you request `dialog`, acss-kit checks whether `button` and `overlay` exist. Missing dependencies are generated first (bottom-up), never silently skipped.

**Atomic WCAG validation.** Theme edits are pre-validated in a temp directory. If any contrast pair fails, the entire batch rolls back. Paired roles (`primary` + `primary-hover`) are always updated together.

**CSS-variable-first.** Every component token â€” `--btn-bg`, `--card-radius`, `--dialog-overlay` â€” is a CSS custom property with a hardcoded fallback. Override anything in your own CSS without touching the generated file.

## Available components

Components are organized into six categories:

| Category         | Components                                                               |
| ---------------- | ------------------------------------------------------------------------ |
| **Simple**       | Badge, Tag, Heading, Text, Link, List, Icon, Img                         |
| **Interactive**  | Button, IconButton                                                       |
| **Form**         | Field, Input, Checkbox                                                   |
| **Layout**       | Card, Nav                                                                |
| **Complex**      | Alert, Dialog, Popover, Table                                            |
| **Form (skill)** | Auto-generated by the `component-form` skill on natural-language prompts |
