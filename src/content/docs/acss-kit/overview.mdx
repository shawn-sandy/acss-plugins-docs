**acss-kit** is a Claude Code plugin that generates accessible React components and CSS themes for [fpkit/acss](https://github.com/shawn-sandy/acss) projects.

## What it generates

<CardGrid>
  <Card title="Components" icon="puzzle">
    Self-contained `.tsx` + `.scss` file pairs written to
    `src/components/fpkit/`. No runtime npm dependency — you own the source.
  </Card>
  <Card title="Themes" icon="seti:css">
    OKLCH-derived CSS custom property files in `src/styles/theme/`. 18 semantic
    color roles, WCAG AA validated, light + dark.
  </Card>
</CardGrid>

## Commands

| Command                                                                        | What it does                                                                                            |
| ------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------- |
| [`/setup`](/acss-plugins-docs/acss-kit/commands/setup)                         | Bootstrap a project — install `ui.tsx`, configure the target, seed a starter theme                      |
| [`/kit-add`](/acss-plugins-docs/acss-kit/commands/kit-add)                     | Generate one or more components (with dependency resolution)                                            |
| [`/kit-add-html`](/acss-plugins-docs/acss-kit/commands/kit-add-html)           | Removed — command file deleted upstream in v1.2.x. Use `/kit-add --target=html` for static HTML output. |
| [`/kit-create`](/acss-plugins-docs/acss-kit/commands/kit-create)               | Generate a single component from a natural-language description                                         |
| [`/kit-list`](/acss-plugins-docs/acss-kit/commands/kit-list)                   | Browse the component catalogue or inspect a specific component                                          |
| [`/kit-sync`](/acss-plugins-docs/acss-kit/commands/kit-sync)                   | Bulk-install every component, foundation, and starter theme in one shot                                 |
| [`/kit-update`](/acss-plugins-docs/acss-kit/commands/kit-update)               | Re-copy unmodified components after a plugin upgrade, preserving your edits                             |
| [`/color-scale`](/acss-plugins-docs/acss-kit/commands/color-scale)             | Generate a 10-step OKLCH color scale from a hex, CSS named color, or theme role                         |
| [`/theme-create`](/acss-plugins-docs/acss-kit/commands/theme-create)           | Generate a full light/dark theme from a seed hex color                                                  |
| [`/theme-brand`](/acss-plugins-docs/acss-kit/commands/theme-brand)             | Scaffold a brand-specific theme override file                                                           |
| [`/theme-extract`](/acss-plugins-docs/acss-kit/commands/theme-extract)         | Extract brand colors from an image or Figma URL (pixel sampling)                                        |
| [`/theme-from-figma`](/acss-plugins-docs/acss-kit/commands/theme-from-figma)   | Generate a theme from a Figma file's variables via the Figma MCP server                                 |
| [`/theme-from-design`](/acss-plugins-docs/acss-kit/commands/theme-from-design) | Generate a full theme from a Google DESIGN.md file                                                      |
| [`/design-export`](/acss-plugins-docs/acss-kit/commands/design-export)         | Export the project's theme tokens to a DESIGN.md (or DTCG/Tailwind via `--format`)                      |
| [`/theme-update`](/acss-plugins-docs/acss-kit/commands/theme-update)           | Edit specific color roles in an existing theme file                                                     |
| [`/style-tune`](/acss-plugins-docs/acss-kit/commands/style-tune)               | Tune theme roles or component tokens with natural-language modifiers                                    |
| [`/prompt-book`](/acss-plugins-docs/acss-kit/commands/prompt-book)             | Print the copy-paste prompt catalogue for all slash commands                                            |
| [`/utility-add`](/acss-plugins-docs/acss-kit/commands/utility-add)             | Copy `utilities.css` + `token-bridge.css` into your project (absorbed from acss-utilities v1.0.0)       |
| [`/utility-bridge`](/acss-plugins-docs/acss-kit/commands/utility-bridge)       | Regenerate `token-bridge.css` against the active theme (absorbed from acss-utilities v1.0.0)            |
| [`/utility-list`](/acss-plugins-docs/acss-kit/commands/utility-list)           | List utility families and classes (absorbed from acss-utilities v1.0.0)                                 |
| [`/utility-tune`](/acss-plugins-docs/acss-kit/commands/utility-tune)           | Adjust spacing, breakpoints, and family settings via natural language (absorbed from acss-utilities)    |

## Skills

As of v1.2.0, per-component generation routes to a dedicated `component-<name>` skill for each component. The `components` skill was renamed to `kit-core` and now handles orchestration only.

| Skill                    | Triggered by                                                        | Purpose                                                                                                                                 |
| ------------------------ | ------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| `setup`                  | `/setup`                                                            | One-shot project bootstrap (ui.tsx, target, optional seed theme)                                                                        |
| `kit-core`               | `/kit-create`, `/kit-list`, `/kit-sync`, `/kit-update`, Form Mode   | Orchestrator — Creator Mode, Form Mode, HTML/Style-Tune modes (renamed from `components` in v1.2.0)                                     |
| `component-<name>` (×15) | `/kit-add <name>`, `/kit-create <name>`                             | Per-component generation (alert, button, card, checkbox, dialog, field, icon, icon-button, img, input, link, list, nav, popover, table) |
| `styles`                 | All `/theme-*` commands                                             | OKLCH palette math, WCAG validation, role catalogue                                                                                     |
| `style-tune`             | `/style-tune` or natural language ("warmer button")                 | Adjusts visual feel via token edits + WCAG validation                                                                                   |
| `kit-sync`               | `/kit-sync`, `/kit-update`                                          | Bulk install and safe re-copy using a SHA256 manifest                                                                                   |
| `utilities`              | `/utility-add`, `/utility-bridge`, `/utility-list`, `/utility-tune` | Atomic CSS utility classes and token bridge (absorbed from acss-utilities v1.0.0)                                                       |
| `prompt-book`            | `/prompt-book`                                                      | Displays the copy-paste prompt catalogue for all slash commands                                                                         |

## Key design decisions

**Vendored source.** Components are written as source files into your project. There's no runtime `@fpkit/acss` import — your bundle doesn't grow, and you can edit the generated files freely.

**Generation contract.** Before writing any file, `/kit-add` shows a preview listing exactly which files will be created or skipped. You always know what's about to happen.

**Dependency resolution.** If you request `dialog`, acss-kit checks whether `button` and `overlay` exist. Missing dependencies are generated first (bottom-up), never silently skipped.

**Atomic WCAG validation.** Theme edits are pre-validated in a temp directory. If any contrast pair fails, the entire batch rolls back. Paired roles (`primary` + `primary-hover`) are always updated together.

**CSS-variable-first.** Every component token — `--btn-bg`, `--card-radius`, `--dialog-overlay` — is a CSS custom property with a hardcoded fallback. Override anything in your own CSS without touching the generated file.

Components are organized into six categories:

| Category        | Components                                                                                                       |
| --------------- | ---------------------------------------------------------------------------------------------------------------- |
| **Simple**      | Badge, Tag, Heading, Text, Link, List, Icon, Img                                                                 |
| **Interactive** | Button, IconButton                                                                                               |
| **Form fields** | Field, Input, Checkbox                                                                                           |
| **Layout**      | Card, Nav                                                                                                        |
| **Complex**     | Alert, Dialog, Popover, Table                                                                                    |
| **Form**        | Generated on-demand via `/kit-create` with a natural-language form description (handled by `kit-core` Form Mode) |

→ See the [Component Catalogue](/acss-plugins-docs/acss-kit/component-catalogue) for the full list with props and CSS tokens.
