The `component-checkbox` skill generates an fpkit-style accessible Checkbox component directly into a developer's project. It is invoked by [`/kit-add checkbox`](/acss-plugins-docs/acss-kit/commands/kit-add) or [`/kit-create`](/acss-plugins-docs/acss-kit/commands/kit-create).

## Workflow

<Steps>

1. **Read `reference.md`** in the skill directory for canonical templates, Generation Contract, Props Interface, CSS Variables, Accessibility requirements, and Usage Examples.

2. **Init check** — runs `detect_target.py` to verify `ui.tsx` is present. If missing, runs `kit-core` Step A initialization first.

3. **Dependencies** — Checkbox depends on `input`. Before generating Checkbox, the skill checks whether `<targetDir>/input.tsx` already exists. If missing, it invokes `component-input` first.

4. **Generate** — writes `checkbox.tsx` and `checkbox.module.scss` to the configured target directory.

5. **Verify** — runs `verify_integration.py` and prints the integration summary.

</Steps>

| `--chk-size`   | Indicator size  | `1rem`                          |

## Accessibility

- `aria-checked` tracks three states: `"true"`, `"false"`, `"mixed"` (indeterminate).
- `aria-disabled="true"` pattern preserves focusability when `disabled={true}`.
- `aria-required` is set when `required={true}`.
- Error message is associated via `aria-describedby`.

## Usage example

```tsx
<Checkbox
  label="I agree to the terms of service"
  checked={agreed}
  onChange={(e) => setAgreed(e.target.checked)}
  required
/>
```

<Aside type="note">
  Checkbox automatically generates its `Input` dependency if `input.tsx` is not
  already present in the target directory — no manual step required.
</Aside>
