The `component-link` skill generates an fpkit-style accessible Link component directly into a developer's project. It is invoked by [`/kit-add link`](/acss-plugins-docs/acss-kit/commands/kit-add), [`/kit-create`](/acss-plugins-docs/acss-kit/commands/kit-create) (when asking for a link), or by calling the `component-link` skill by name.

|                  |                                                                                                                 |
| ---------------- | --------------------------------------------------------------------------------------------------------------- |
| **Triggered by** | `/kit-add link`, `/kit-create` → "link", or direct skill invocation                                             |
| **Output**       | `<targetDir>/link.tsx` + `<targetDir>/link.module.scss`                                                         |
| **Dependencies** | None — leaf component                                                                                           |
| **Hint**         | Describe whether links open in a new tab, need external detection, or require current-page aria-current marking |

## Workflow

<Steps>

1. **Read** `reference.md` in the `component-link` skill directory for the canonical TSX/SCSS templates, Generation Contract, Props Interface, CSS Variables, Accessibility requirements, and Usage Examples.

2. **Init check** — run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/detect_target.py <cwd>`. If the foundation (`ui.tsx`) is missing, run Step A of the [`kit-core` skill](/acss-plugins-docs/acss-kit/skills/kit-core) to initialize it before generating this component.

3. **Dependencies** — Link has no upstream dependencies. Skip dep resolution.

4. **Generate** — apply the Generation Contract from `reference.md` to produce:
   - `<targetDir>/link.tsx` from the `## TSX Template` section
   - `<targetDir>/link.module.scss` from the `## SCSS Template` section

5. **Verify** — run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/verify_integration.py <projectRoot>` and print the summary to the developer.

</Steps>

## Invocation hints

```text
/kit-add link
```

```text
/kit-create — "create a link component that adds rel=noopener for external URLs"
```

## CSS variable prefix

Link tokens follow the `--link-*` naming convention, e.g. `--link-color`, `--link-color-hover`, `--link-underline-offset`.

## Accessibility

External links that open in a new tab automatically append visually-hidden text ("opens in new tab") and set `rel="noopener noreferrer"`. Current-page links use `aria-current="page"`.
