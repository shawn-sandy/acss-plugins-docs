The `component-link` skill generates an fpkit-style accessible Link component directly into a developer's project. It is invoked by [`/kit-add link`](/acss-plugins-docs/acss-kit/commands/kit-add) or [`/kit-create`](/acss-plugins-docs/acss-kit/commands/kit-create).

## Workflow

<Steps>

1. **Read `reference.md`** in the skill directory for canonical templates, Generation Contract, Props Interface, CSS Variables, Accessibility requirements, and Usage Examples.

2. **Init check** ‚Äî runs `detect_target.py` to verify `ui.tsx` is present. If missing, runs `kit-core` Step A initialization first.

3. **Dependencies** ‚Äî Link has no upstream component dependencies (it is a leaf component). Dependency resolution is skipped.

4. **Generate** ‚Äî writes `link.tsx` and `link.module.scss` to the configured target directory.

5. **Verify** ‚Äî runs `verify_integration.py` and prints the integration summary.

</Steps>

## External link handling

When `href` starts with `http://` or `https://`, or when `external={true}` is set, the component automatically:

- Adds `target="_blank"` and `rel="noopener noreferrer"`.
- Appends a visually-hidden `" (opens in new tab)"` string for screen readers.
- Optionally renders an external-link icon (decorative, `aria-hidden="true"`).

## Props

| Prop        | Type        | Required | Default       |
| ----------- | ----------- | -------- | ------------- |
| `href`      | `string`    | Yes      | ‚Äî             |
| `children`  | `ReactNode` | Yes      | ‚Äî             |
| `external`  | `boolean`   | No       | auto-detected |
| `current`   | `boolean`   | No       | `false`       |
| `label`     | `string`    | No       | ‚Äî             |
| `className` | `string`    | No       | ‚Äî             |

## CSS tokens

| Token              | Role             | Fallback                        |
| ------------------ | ---------------- | ------------------------------- |
| `--link-color`     | Default color    | `var(--color-primary, #4f46e5)` |
| `--link-visited`   | Visited color    | `var(--color-secondary)`        |
| `--link-hover`     | Hover color      | `var(--color-primary-dark)`     |
| `--link-underline` | Underline offset | `2px`                           |

## Accessibility

- External links get `rel="noopener noreferrer"` and a visually-hidden new-tab warning.
- `aria-current="page"` is set when `current={true}`.
- `aria-label` overrides the accessible name when `label` is provided ‚Äî use sparingly.
- `:focus-visible` styles use `--color-focus-ring`.

## Usage example

```tsx
<Link href="/acss-plugins-docs/getting-started/introduction">Get started</Link>
<Link href="https://github.com/shawn-sandy/agentic-acss-plugins" external>
  View on GitHub
</Link>
<Link href="/acss-plugins-docs/acss-kit/overview" current>acss-kit</Link>
```
