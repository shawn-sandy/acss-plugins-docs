The `component-link` skill generates a self-contained accessible Link component directly into a developer's project. It is invoked via `/kit-add link` or `/kit-create` (then ask for a link).

<Aside type="note">
  This skill was added in acss-kit v1.2.0. It sets `disable-model-invocation:
  true` so it stays out of session auto-context. Invoke it explicitly via
  `/kit-add link`, `/kit-create`, or by calling `component-link` by name.
</Aside>

|                  |                                                                                                                                         |
| ---------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| **Triggered by** | `/kit-add link`, `/kit-create` (link description), direct `component-link` skill name                                                   |
| **Output**       | `link.tsx` + `link.module.scss`                                                                                                         |
| **Dependencies** | None (leaf component)                                                                                                                   |
| **Hint**         | Describe the href, the link text, whether it opens in a new tab/external (so rel and aria-label can be set), and any current-page state |

## Accessibility

- External links automatically receive `rel="noopener noreferrer"` and an `aria-label` suffix indicating they open in a new tab (e.g. `"Visit example.com (opens in new tab)"`).
- `aria-current="page"` applied when the `current` prop is set, indicating the current page in navigation contexts.
- `:focus-visible` outline preserved in SCSS using `--color-focus-ring`.
