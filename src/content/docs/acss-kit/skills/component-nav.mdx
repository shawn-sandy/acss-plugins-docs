The `component-nav` skill generates an fpkit-style accessible Nav component directly into a developer's project. It is invoked by [`/kit-add nav`](/acss-plugins-docs/acss-kit/commands/kit-add) or [`/kit-create`](/acss-plugins-docs/acss-kit/commands/kit-create).

## Workflow

<Steps>

1. **Read `reference.md`** in the skill directory for canonical templates, Generation Contract, Props Interface, CSS Variables, Accessibility requirements, and Usage Examples.

2. **Init check** ‚Äî runs `detect_target.py` to verify `ui.tsx` is present. If missing, runs `kit-core` Step A initialization first.

3. **Dependencies** ‚Äî Nav has no upstream component dependencies (it is a leaf component). Dependency resolution is skipped.

4. **Generate** ‚Äî writes `nav.tsx` and `nav.module.scss` to the configured target directory.

5. **Verify** ‚Äî runs `verify_integration.py` and prints the integration summary.

</Steps>

## Props

| Prop          | Type                         | Required | Default        |
| ------------- | ---------------------------- | -------- | -------------- |
| `label`       | `string`                     | Yes      | ‚Äî              |
| `items`       | `NavItem[]`                  | Yes      | ‚Äî              |
| `layout`      | `"horizontal" \| "vertical"` | No       | `"horizontal"` |
| `currentHref` | `string`                     | No       | ‚Äî              |
| `className`   | `string`                     | No       | ‚Äî              |

`NavItem` shape:

```ts
interface NavItem {
  label: string;
  href: string;
}
```

## CSS tokens

| Token                | Role               | Fallback                          |
| -------------------- | ------------------ | --------------------------------- |
| `--nav-bg`           | Nav background     | `var(--color-surface, #fff)`      |
| `--nav-gap`          | Item spacing       | `0.25rem`                         |
| `--nav-item-color`   | Link color         | `var(--color-text, #111)`         |
| `--nav-active-bg`    | Current-page bg    | `var(--color-primary, #4f46e5)`   |
| `--nav-active-color` | Current-page color | `var(--color-text-inverse, #fff)` |
| `--nav-radius`       | Item border radius | `var(--radius-md, 0.375rem)`      |

## Accessibility

- Renders a `<nav>` landmark ‚Äî `aria-label` is **always required** to distinguish multiple nav landmarks on the same page.
- `aria-current="page"` is applied to the item matching `currentHref`.
- All link items are `<a>` elements ‚Äî no `<button>` inside a nav unless the item triggers a non-navigation action.

<Aside type="caution">
  The `label` prop is required. Without it, screen reader users cannot
  distinguish this navigation region from others on the page (WCAG 2.4.1).
</Aside>

## Usage example

```tsx
<Nav
  label="Main navigation"
  currentHref={router.pathname}
  items={[
    { label: "Home", href: "/" },
    { label: "Docs", href: "/docs" },
    { label: "Examples", href: "/examples" },
  ]}
/>
```
