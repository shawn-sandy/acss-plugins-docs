The `component-nav` skill generates a self-contained accessible Nav component directly into a developer's project. It is invoked via `/kit-add nav` or `/kit-create` (then ask for a nav).

<Aside type="note">
  This skill was added in acss-kit v1.2.0. It sets `disable-model-invocation:
  true` so it stays out of session auto-context. Invoke it explicitly via
  `/kit-add nav`, `/kit-create`, or by calling `component-nav` by name.
</Aside>

|                  |                                                                                                                                        |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| **Triggered by** | `/kit-add nav`, `/kit-create` (nav description), direct `component-nav` skill name                                                     |
| **Output**       | `nav.tsx` + `nav.module.scss`                                                                                                          |
| **Dependencies** | None (leaf component)                                                                                                                  |
| **Hint**         | Describe the aria-label for the landmark, the link items (label + href), the current-page link, and horizontal vs vertical orientation |

## Workflow

<Steps>

1. **Read the source doc.** Prefer `nav.component.md` in the skill directory (the spec-driven source of truth for the Generation Contract, Props Interface, SCSS, and TSX). Fall back to `reference.md` if the component file is absent. Both sources yield byte-identical output. Note: Nav is legacy-shape тАФ `nav.component.md` has no `## TSX Template` block; the generated `nav.tsx` is assembled from `## Props Interface` and `## Key Pattern: Compound Component Assembly`.

2. **Init check** тАФ run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/detect_target.py <cwd>`. If the foundation (`ui.tsx`) is missing, run Step A of `${CLAUDE_PLUGIN_ROOT}/skills/kit-core/SKILL.md` to initialize it before generating this component.

3. **Dependencies** тАФ Nav has no upstream dependencies. Skip dep resolution.

4. **Generate** тАФ apply the Generation Contract to produce:
   - `<targetDir>/nav.tsx` assembled from `## Props Interface` and `## Key Pattern: Compound Component Assembly` in `nav.component.md` (or the `## TSX Template` section in `reference.md`)
   - `<targetDir>/nav.module.scss` from the `## Styles` section in `nav.component.md` (or `## SCSS Template` in `reference.md`)

5. **Verify** тАФ run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/verify_integration.py <projectRoot>` and print the summary.

</Steps>

## Accessibility

- `<nav>` landmark element with a required `aria-label` to distinguish multiple nav regions on a page.
- `aria-current="page"` applied to the active link item.
- Horizontal and vertical layout variants controlled via a `direction` prop and CSS custom properties.
- Each link inherits `:focus-visible` focus ring from the shared `--color-focus-ring` token.
