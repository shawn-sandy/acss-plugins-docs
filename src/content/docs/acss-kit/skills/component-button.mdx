The `component-button` skill generates a self-contained, production-quality `Button.tsx` + `Button.scss` pair directly into your project. It is invoked by `/kit-add button` and by `/kit-create` when the description names a button.

## Trigger

| Invocation                     | Notes                                                                      |
| ------------------------------ | -------------------------------------------------------------------------- |
| `/kit-add button`              | Standard generation workflow                                               |
| `/kit-create primary button …` | Creator Mode via [`kit-core`](/acss-plugins-docs/acss-kit/skills/kit-core) |

The skill uses `disable-model-invocation: true` — it does **not** auto-trigger on general conversation. You must invoke it explicitly via a slash command or by name.

## Workflow

<Steps>

1. **Read** `reference.md` in the skill directory for the canonical TSX template, generation contract, Props Interface, CSS Variables, accessibility requirements, and usage examples.

2. **Init check** — run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/detect_target.py <cwd>`. If `ui.tsx` (the foundation) is missing, run Step A of `kit-core/SKILL.md` to initialize it before generating this component.

3. **Dependency tree** — `Button` has no external component dependencies. The skill writes `Button.tsx` and `Button.scss` only.

4. **Show the plan** — display every file that will be created (and any that will be skipped) before writing anything.

5. **Generate** — write `Button.tsx` and `Button.scss` to the target directory (default `src/components/fpkit/`).

6. **Summary** — print the created/skipped file list and the import statement to add to your code.

</Steps>

## Generation contract

| File          | Output path (default)              |
| ------------- | ---------------------------------- |
| `Button.tsx`  | `src/components/fpkit/Button.tsx`  |
| `Button.scss` | `src/components/fpkit/Button.scss` |

No component dependencies. `ui.tsx` must be present (copied by the init check if absent).

## Props

| Prop        | Type                                           | Required | Default   | Description                                      |
| ----------- | ---------------------------------------------- | -------- | --------- | ------------------------------------------------ |
| `children`  | `ReactNode`                                    | Yes      | —         | Button label or content                          |
| `variant`   | `primary \| secondary \| ghost \| destructive` | No       | `primary` | Visual treatment                                 |
| `size`      | `sm \| md \| lg`                               | No       | `md`      | Size scale                                       |
| `disabled`  | `boolean`                                      | No       | `false`   | Accessible disabled state (uses `aria-disabled`) |
| `onClick`   | `MouseEventHandler`                            | No       | —         | Click handler (suppressed when disabled)         |
| `className` | `string`                                       | No       | —         | Additional CSS classes                           |

## CSS tokens

| Token               | Role               | Fallback                          |
| ------------------- | ------------------ | --------------------------------- |
| `--btn-bg`          | Background color   | `var(--color-primary, #4f46e5)`   |
| `--btn-color`       | Text color         | `var(--color-text-inverse, #fff)` |
| `--btn-border`      | Border color       | `transparent`                     |
| `--btn-radius`      | Border radius      | `var(--radius-md, 0.375rem)`      |
| `--btn-padding-x`   | Horizontal padding | `1rem`                            |
| `--btn-padding-y`   | Vertical padding   | `0.5rem`                          |
| `--btn-font-size`   | Font size          | `1rem`                            |
| `--btn-font-weight` | Font weight        | `600`                             |

## Accessibility

- Uses `aria-disabled="true"` instead of the `disabled` attribute — keeps the element keyboard-focusable and announced by screen readers.
- `onClick` is suppressed when `disabled` is `true` via the `useDisabledState` hook.
- `:focus-visible` styles use `--color-focus-ring`.
