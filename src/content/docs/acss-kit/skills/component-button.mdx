The `component-button` skill generates an fpkit-style accessible Button component directly into a developer's project. It is invoked by [`/kit-add button`](/acss-plugins-docs/acss-kit/commands/kit-add) or [`/kit-create`](/acss-plugins-docs/acss-kit/commands/kit-create).

## Workflow

<Steps>

1. **Read `reference.md`** in the skill directory for canonical templates, Generation Contract, Props Interface, CSS Variables, Accessibility requirements, and Usage Examples.

2. **Init check** ‚Äî runs `detect_target.py` to verify `ui.tsx` is present. If missing, runs `kit-core` Step A initialization first.

3. **Dependencies** ‚Äî Button has no upstream component dependencies (it is a leaf component). Dependency resolution is skipped.

4. **Generate** ‚Äî writes `button.tsx` and `button.module.scss` to the configured target directory using the TSX/SCSS templates from `reference.md`.

5. **Verify** ‚Äî runs `verify_integration.py` and prints the integration summary.

</Steps>

## Props

| Prop        | Type                                              | Required | Default     |
| ----------- | ------------------------------------------------- | -------- | ----------- |
| `children`  | `ReactNode`                                       | Yes      | ‚Äî           |
| `variant`   | `"primary" \| "secondary" \| "ghost" \| "danger"` | No       | `"primary"` |
| `size`      | `"sm" \| "md" \| "lg"`                            | No       | `"md"`      |
| `disabled`  | `boolean`                                         | No       | `false`     |
| `onClick`   | `MouseEventHandler`                               | No       | ‚Äî           |
| `className` | `string`                                          | No       | ‚Äî           |

## CSS tokens

| Token               | Role               | Fallback                          |
| ------------------- | ------------------ | --------------------------------- |
| `--btn-bg`          | Background color   | `var(--color-primary, #4f46e5)`   |
| `--btn-color`       | Text color         | `var(--color-text-inverse, #fff)` |
| `--btn-border`      | Border color       | `transparent`                     |
| `--btn-radius`      | Border radius      | `var(--radius-md, 0.375rem)`      |
| `--btn-padding-x`   | Horizontal padding | `1rem`                            |
| `--btn-padding-y`   | Vertical padding   | `0.5rem`                          |
| `--btn-font-size`   | Font size          | `1rem`                            |
| `--btn-font-weight` | Font weight        | `600`                             |

## Accessibility

- Uses `aria-disabled="true"` instead of the native `disabled` attribute ‚Äî keeps the button focusable and announced by screen readers (WCAG 2.1.1).
- Provides `:focus-visible` styles using `--color-focus-ring`.
- `data-variant` drives visual state ‚Äî no extra class juggling needed.

## Usage example

```tsx
<Button variant="primary" onClick={handleSave}>Save changes</Button>
<Button variant="secondary">Cancel</Button>
<Button variant="danger" disabled>Delete account</Button>
```

  Use `/kit-list button` to preview the full generation contract, props, and CSS
  tokens before running `/kit-add button`.
</Aside>
