The `component-table` skill generates a self-contained accessible Table component directly into a developer's project. It is invoked via `/kit-add table` or `/kit-create` (then ask for a table).

<Aside type="note">
  This skill was added in acss-kit v1.2.0. It sets `disable-model-invocation:
  true` so it stays out of session auto-context. Invoke it explicitly via
  `/kit-add table`, `/kit-create`, or by calling `component-table` by name.
</Aside>

|                  |                                                                                                                                                      |
| ---------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Triggered by** | `/kit-add table`, `/kit-create` (table description), direct `component-table` skill name                                                             |
| **Output**       | `table.tsx` + `table.module.scss`                                                                                                                    |
| **Dependencies** | None (leaf component)                                                                                                                                |
| **Hint**         | Describe the caption, the column headers (with scope), the row shape, whether any columns are sortable, and if a responsive scroll wrapper is needed |

## Workflow

<Steps>

1. **Read the source doc.** Prefer `table.component.md` in the skill directory (the spec-driven source of truth for the Generation Contract, Props Interface, SCSS, and TSX). Fall back to `reference.md` if the component file is absent. Both sources yield byte-identical output (golden-guarded).

2. **Init check** — run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/detect_target.py <cwd>`. If the foundation (`ui.tsx`) is missing, run Step A of `${CLAUDE_PLUGIN_ROOT}/skills/kit-core/SKILL.md` to initialize it before generating this component.

3. **Dependencies** — Table has no upstream dependencies. Skip dep resolution.

4. **Generate** — apply the Generation Contract to produce:
   - `<targetDir>/table.tsx` from the `## Target: react` adapter in `table.component.md` (or the top-level `## TSX Template` section in `reference.md`)
   - `<targetDir>/table.module.scss` from the `## Styles` section in `table.component.md` (or `## SCSS Template` in `reference.md`)

5. **Verify** — run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/verify_integration.py <projectRoot>` and print the summary.

</Steps>

## Accessibility

- `<caption>` is always rendered — visible by default, hideable via `sr-only` class for screen-reader-only descriptions.
- Column headers use `scope="col"` and row headers use `scope="row"` to associate cells correctly.
- Sortable columns add `aria-sort="ascending"` / `"descending"` / `"none"` and a sort-trigger button inside the `<th>`.
- Responsive scroll wrapper (`overflow-x: auto`) uses `role="region"` with `aria-label` so keyboard users can scroll it via the Tab key.
