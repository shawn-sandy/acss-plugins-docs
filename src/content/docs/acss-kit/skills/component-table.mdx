The `component-table` skill generates an fpkit-style accessible Table component directly into a developer's project. It is invoked by [`/kit-add table`](/acss-plugins-docs/acss-kit/commands/kit-add), [`/kit-create`](/acss-plugins-docs/acss-kit/commands/kit-create) (when asking for a table), or by calling the `component-table` skill by name.

|                  |                                                                                                                     |
| ---------------- | ------------------------------------------------------------------------------------------------------------------- |
| **Triggered by** | `/kit-add table`, `/kit-create` → "table", or direct skill invocation                                               |
| **Output**       | `<targetDir>/table.tsx` + `<targetDir>/table.module.scss`                                                           |
| **Dependencies** | None — leaf component                                                                                               |
| **Hint**         | Describe whether you need a caption, sortable columns, a responsive scroll wrapper, and whether rows are selectable |

## Workflow

<Steps>

1. **Read** `reference.md` in the `component-table` skill directory for the canonical TSX/SCSS templates, Generation Contract, Props Interface, CSS Variables, Accessibility requirements, and Usage Examples.

2. **Init check** — run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/detect_target.py <cwd>`. If the foundation (`ui.tsx`) is missing, run Step A of the [`kit-core` skill](/acss-plugins-docs/acss-kit/skills/kit-core) to initialize it before generating this component.

3. **Dependencies** — Table has no upstream dependencies. Skip dep resolution.

4. **Generate** — apply the Generation Contract from `reference.md` to produce:
   - `<targetDir>/table.tsx` from the `## TSX Template` section
   - `<targetDir>/table.module.scss` from the `## SCSS Template` section

5. **Verify** — run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/verify_integration.py <projectRoot>` and print the summary to the developer.

</Steps>

## Invocation hints

```text
/kit-add table
```

```text
/kit-create — "create a sortable data table with caption and responsive scroll wrapper"
```

## CSS variable prefix

Table tokens follow the `--table-*` naming convention, e.g. `--table-border`, `--table-row-bg-alt`, `--table-header-bg`.

## Accessibility

Table includes a `<caption>` element (required for screen readers). Column headers use `scope="col"` and row headers use `scope="row"`. Sortable columns expose `aria-sort="ascending"` / `"descending"` / `"none"` on the `<th>` element.
