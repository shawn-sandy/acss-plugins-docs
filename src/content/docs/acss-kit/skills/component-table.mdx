## Workflow

<Steps>

1. **Read `reference.md`** in the skill directory for canonical templates, Generation Contract, Props Interface, CSS Variables, Accessibility requirements, and Usage Examples.

2. **Init check** ‚Äî runs `detect_target.py` to verify `ui.tsx` is present. If missing, runs `kit-core` Step A initialization first.

3. **Dependencies** ‚Äî Table has no upstream component dependencies (it is a leaf component). Dependency resolution is skipped.

4. **Generate** ‚Äî writes `table.tsx` and `table.module.scss` to the configured target directory.

5. **Verify** ‚Äî runs `verify_integration.py` and prints the integration summary.

</Steps>

## Column definition

Columns are defined as an array of `ColumnDef` objects:

```ts
interface ColumnDef<T> {
  key: keyof T;
  header: string;
  sortable?: boolean;
  render?: (value: T[keyof T], row: T) => ReactNode;
}
```

## Props

| Prop        | Type                     | Required | Default |
| ----------- | ------------------------ | -------- | ------- |
| `columns`   | `ColumnDef<T>[]`         | Yes      | ‚Äî       |
| `rows`      | `T[]`                    | Yes      | ‚Äî       |
| `caption`   | `string`                 | Yes      | ‚Äî       |
| `sortKey`   | `keyof T`                | No       | ‚Äî       |
| `sortDir`   | `"asc" \| "desc"`        | No       | `"asc"` |
| `onSort`    | `(key: keyof T) => void` | No       | ‚Äî       |
| `className` | `string`                 | No       | ‚Äî       |

## CSS tokens

| Token               | Role               | Fallback                       |
| ------------------- | ------------------ | ------------------------------ |
| `--tbl-border`      | Cell border color  | `var(--color-border, #e5e7eb)` |
| `--tbl-head-bg`     | Header background  | `var(--color-surface, #fff)`   |
| `--tbl-head-weight` | Header font weight | `600`                          |
| `--tbl-row-hover`   | Row hover bg       | `var(--color-surface-raised)`  |
| `--tbl-padding-x`   | Cell h-padding     | `0.75rem`                      |
| `--tbl-padding-y`   | Cell v-padding     | `0.5rem`                       |
| `--tbl-radius`      | Outer radius       | `var(--radius-md, 0.375rem)`   |

## Accessibility

- `<caption>` is always required ‚Äî it provides the table's accessible name.
- All column headers use `scope="col"` and all row headers use `scope="row"`.
- Sortable columns set `aria-sort="ascending"` or `aria-sort="descending"` on the active header; inactive sortable headers get `aria-sort="none"`.
- The table is wrapped in a `<div>` with `overflow-x: auto` for responsive scrolling; the scroll container has `tabindex="0"` so keyboard users can scroll it.

<Aside type="caution">
  The `caption` prop is required. A table without a caption has no accessible
  name ‚Äî screen reader users cannot distinguish it from other tables on the
  page.
</Aside>

## Usage example

```tsx
<Table
  caption="Team members"
  columns={[
    { key: "name", header: "Name", sortable: true },
    { key: "role", header: "Role" },

  sortKey={sortKey}
```
