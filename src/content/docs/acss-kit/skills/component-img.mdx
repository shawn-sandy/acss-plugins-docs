The `component-img` skill generates an fpkit-style accessible Img component directly into a developer's project. It is invoked by [`/kit-add img`](/acss-plugins-docs/acss-kit/commands/kit-add), [`/kit-create`](/acss-plugins-docs/acss-kit/commands/kit-create) (when asking for an image component), or by calling the `component-img` skill by name.

|                  |                                                                                                                                        |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| **Triggered by** | `/kit-add img`, `/kit-create` → "image", or direct skill invocation                                                                    |
| **Output**       | `<targetDir>/img.tsx` + `<targetDir>/img.module.scss`                                                                                  |
| **Dependencies** | None — leaf component                                                                                                                  |
| **Hint**         | Describe whether images are decorative (empty alt) or semantic (descriptive alt), the aspect ratio, and whether lazy-loading is needed |

## Workflow

<Steps>

1. **Read** `reference.md` in the `component-img` skill directory for the canonical TSX/SCSS templates, Generation Contract, Props Interface, CSS Variables, Accessibility requirements, and Usage Examples.

2. **Init check** — run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/detect_target.py <cwd>`. If the foundation (`ui.tsx`) is missing, run Step A of the [`kit-core` skill](/acss-plugins-docs/acss-kit/skills/kit-core) to initialize it before generating this component.

3. **Dependencies** — Img has no upstream dependencies. Skip dep resolution.

4. **Generate** — apply the Generation Contract from `reference.md` to produce:
   - `<targetDir>/img.tsx` from the `## TSX Template` section
   - `<targetDir>/img.module.scss` from the `## SCSS Template` section

5. **Verify** — run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/verify_integration.py <projectRoot>` and print the summary to the developer.

</Steps>

## Invocation hints

```text
/kit-add img
```

```text
/kit-create — "create an image component with 16/9 aspect ratio and lazy loading"
```

## CSS variable prefix

Img tokens follow the `--img-*` naming convention, e.g. `--img-radius`, `--img-aspect-ratio`.

## Accessibility

Decorative images use `alt=""`. All other images require a descriptive `alt` prop — enforced by the TypeScript interface. `loading="lazy"` is applied by default.
