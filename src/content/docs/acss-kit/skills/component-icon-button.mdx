The `component-icon-button` skill generates a self-contained accessible IconButton component directly into a developer's project. It is invoked via `/kit-add icon-button` or `/kit-create` (then ask for an icon button).

<Aside type="note">
  This skill was added in acss-kit v1.2.0. It sets `disable-model-invocation:
  true` so it stays out of session auto-context. Invoke it explicitly via
  `/kit-add icon-button`, `/kit-create`, or by calling `component-icon-button`
  by name.
</Aside>

|                  |                                                                                                                            |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------- |
| **Triggered by** | `/kit-add icon-button`, `/kit-create` (icon button description), direct `component-icon-button` skill name                 |
| **Output**       | `icon-button.tsx` + `icon-button.module.scss`                                                                              |
| **Dependencies** | `button` â€” generates Button first if missing                                                                               |
| **Hint**         | Describe the icon, the required aria-label (always required for icon-only buttons), the variant/size, and any tooltip text |

3. **Dependencies** â€” IconButton depends on `button`. Before generating IconButton, check whether `<targetDir>/button.tsx` already exists. If missing, invoke the `component-button` skill first, then continue.

4. **Generate** â€” apply the Generation Contract to produce:
   - `<targetDir>/icon-button.tsx` from the `## TSX Template` (in `icon-button.component.md`, or top-level in `reference.md`)

5. **Verify** â€” run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/verify_integration.py <projectRoot>` and print the summary.

</Steps>

## Accessibility

- `aria-label` is **always required** for icon-only buttons â€” the skill halts with an error if it is missing.
- Tooltip fallback uses `title` attribute as a secondary label for pointer users.
- Inherits the `aria-disabled` pattern from the Button dependency (WCAG 2.1.1).
