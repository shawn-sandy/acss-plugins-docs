## Workflow

<Steps>

1. **Read `reference.md`** in the skill directory for canonical templates, Generation Contract, Props Interface, CSS Variables, Accessibility requirements, and Usage Examples.

2. **Init check** — runs `detect_target.py` to verify `ui.tsx` is present. If missing, runs `kit-core` Step A initialization first.

3. **Dependencies** — IconButton depends on `button`. Before generating IconButton, the skill checks whether `<targetDir>/button.tsx` already exists. If missing, it invokes `component-button` first.

4. **Generate** — writes `icon-button.tsx` and `icon-button.module.scss` to the configured target directory.

5. **Verify** — runs `verify_integration.py` and prints the integration summary.

</Steps>

## Props

| Prop        | Type                                              | Required | Default   |
| ----------- | ------------------------------------------------- | -------- | --------- |
| `label`     | `string`                                          | Yes      | —         |
| `icon`      | `ReactNode`                                       | Yes      | —         |
| `variant`   | `"primary" \| "secondary" \| "ghost" \| "danger"` | No       | `"ghost"` |
| `size`      | `"sm" \| "md" \| "lg"`                            | No       | `"md"`    |
| `disabled`  | `boolean`                                         | No       | `false`   |
| `tooltip`   | `string`                                          | No       | —         |
| `onClick`   | `MouseEventHandler`                               | No       | —         |
| `className` | `string`                                          | No       | —         |

## CSS tokens

Inherits all `--btn-*` tokens from the `Button` component, plus:

| Token               | Role              | Fallback                     |
| ------------------- | ----------------- | ---------------------------- |
| `--icon-btn-size`   | Button dimensions | `2rem`                       |
| `--icon-btn-radius` | Border radius     | `var(--radius-md, 0.375rem)` |

## Accessibility

- `aria-label` is **always required** — icon-only buttons have no visible text, so the label is the only accessible name.
- The inner SVG icon always gets `aria-hidden="true"` to prevent duplicate announcements.
- `aria-disabled="true"` pattern preserves focusability when `disabled={true}`.
- When `tooltip` is provided, the tooltip is connected via `aria-describedby`.

<Aside type="caution">
  The `label` prop is required with no default. Omitting it will cause a
  TypeScript error at compile time and an accessibility violation at runtime.
</Aside>

## Usage example

```tsx
<IconButton label="Close dialog" icon={<CloseIcon />} onClick={handleClose} />
<IconButton label="Account settings" icon={<SettingsIcon />} variant="ghost" />
<IconButton label="Delete item" icon={<TrashIcon />} variant="danger" disabled />
```
