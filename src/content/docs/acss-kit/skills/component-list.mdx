The `component-list` skill generates an fpkit-style accessible List component directly into a developer's project. It is invoked by [`/kit-add list`](/acss-plugins-docs/acss-kit/commands/kit-add), [`/kit-create`](/acss-plugins-docs/acss-kit/commands/kit-create) (when asking for a list), or by calling the `component-list` skill by name.

|                  |                                                                                                             |
| ---------------- | ----------------------------------------------------------------------------------------------------------- |
| **Triggered by** | `/kit-add list`, `/kit-create` ŌåÆ "list", or direct skill invocation                                         |
| **Output**       | `<targetDir>/list.tsx` + `<targetDir>/list.module.scss`                                                     |
| **Dependencies** | None ŌĆö leaf component                                                                                       |
| **Hint**         | Describe whether you need ordered or unordered, inline or stacked layout, and whether list styling is reset |

## Workflow

<Steps>

1. **Read** `reference.md` in the `component-list` skill directory for the canonical TSX/SCSS templates, Generation Contract, Props Interface, CSS Variables, Accessibility requirements, and Usage Examples.

5. **Verify** ŌĆö run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/verify_integration.py <projectRoot>` and print the summary to the developer.

</Steps>

## CSS variable prefix

List tokens follow the `--list-*` naming convention, e.g. `--list-gap`, `--list-item-color`.

## Accessibility

List applies `role="list"` explicitly to restore list semantics when CSS `list-style: none` is used (required for VoiceOver compatibility). Item elements use `role="listitem"`.
