The `component-list` skill generates a self-contained accessible List component directly into a developer's project. It is invoked via `/kit-add list` or `/kit-create` (then ask for a list).

<Aside type="note">
  This skill was added in acss-kit v1.2.0. It sets `disable-model-invocation:
  true` so it stays out of session auto-context. Invoke it explicitly via
  `/kit-add list`, `/kit-create`, or by calling `component-list` by name.
</Aside>

|                  |                                                                                                                                            |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| **Triggered by** | `/kit-add list`, `/kit-create` (list description), direct `component-list` skill name                                                      |
| **Output**       | `list.tsx` + `list.module.scss`                                                                                                            |
| **Dependencies** | None (leaf component)                                                                                                                      |
| **Hint**         | Describe ordered vs unordered, the item content/shape, inline vs stacked layout, and whether the list needs a marker reset (`role="list"`) |

## Workflow

<Steps>

1. **Read** `reference.md` in the skill directory for the canonical TSX/SCSS templates, Generation Contract, Props Interface, CSS Variables, Accessibility requirements, and Usage Examples.

2. **Init check** — run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/detect_target.py <cwd>`. If the foundation (`ui.tsx`) is missing, run Step A of `${CLAUDE_PLUGIN_ROOT}/skills/kit-core/SKILL.md` to initialize it before generating this component.

3. **Dependencies** — List has no upstream dependencies. Skip dep resolution.

4. **Generate** — apply the Generation Contract from `reference.md` to produce:
   - `<targetDir>/list.tsx` from the `## TSX Template` section
   - `<targetDir>/list.module.scss` from the `## SCSS Template` section

5. **Verify** — run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/verify_integration.py <projectRoot>` and print the summary.

</Steps>

## Accessibility

- `role="list"` is applied explicitly when CSS `list-style: none` is used, restoring list semantics in Safari/VoiceOver (see [WebKit bug](https://bugs.webkit.org/show_bug.cgi?id=170179)).
- Supports `<ol>` (ordered) and `<ul>` (unordered) via the `as` prop.
- Item slots accept any React node — text, links, or compound content.
