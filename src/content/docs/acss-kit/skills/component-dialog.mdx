The `component-dialog` skill generates an fpkit-style accessible Dialog component directly into a developer's project. It is invoked by [`/kit-add dialog`](/acss-plugins-docs/acss-kit/commands/kit-add) or [`/kit-create`](/acss-plugins-docs/acss-kit/commands/kit-create).

## Workflow

<Steps>

1. **Read `reference.md`** in the skill directory for canonical templates, Generation Contract, Props Interface, CSS Variables, Accessibility requirements, and Usage Examples.

2. **Init check** ã runs `detect_target.py` to verify `ui.tsx` is present. If missing, runs `kit-core` Step A initialization first.

3. **Dependencies** ã Dialog depends on `button`. Before generating Dialog, the skill checks whether `<targetDir>/button.tsx` already exists. If missing, it invokes `component-button` first.

4. **Generate** ã writes `dialog.tsx` and `dialog.module.scss` to the configured target directory.

5. **Verify** ã runs `verify_integration.py` and prints the integration summary.

</Steps>

## Accessibility

- Uses the native `<dialog>` element with `aria-modal="true"` and `aria-labelledby` pointing to the title.
- Focus is trapped inside the dialog while open; returns to the trigger element on close.
- `Escape` key invokes `onClose`.
- Backdrop click invokes `onClose` when `modal={true}`.

## Usage example

```tsx
<Dialog
  title="Delete account"
  open={isOpen}
  onClose={() => setIsOpen(false)}
  footer={
    <>
      <Button variant="secondary" onClick={() => setIsOpen(false)}>
        Cancel
      </Button>
      <Button variant="danger" onClick={handleDelete}>
        Delete
      </Button>
    </>
  }
>
  This action is permanent and cannot be reversed.
</Dialog>
```

<Aside type="note">
  Dialog automatically generates its `Button` dependency if `button.tsx` is not
  already present in the target directory.
</Aside>
