The `component-dialog` skill generates a self-contained accessible Dialog component directly into a developer's project. It is invoked via `/kit-add dialog` or `/kit-create` (then ask for a dialog).

<Aside type="note">
  This skill was added in acss-kit v1.2.0. It sets `disable-model-invocation:
  true` so it stays out of session auto-context. Invoke it explicitly via
  `/kit-add dialog`, `/kit-create`, or by calling `component-dialog` by name.
</Aside>

|                  |                                                                                                                               |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| **Triggered by** | `/kit-add dialog`, `/kit-create` (dialog description), direct `component-dialog` skill name                                   |
| **Output**       | `dialog.tsx` + `dialog.module.scss`                                                                                           |
| **Dependencies** | `button` ā€” generates Button first if missing                                                                                  |
| **Hint**         | Describe the title, modal vs non-modal, the trigger element, the body content, and any footer action buttons (confirm/cancel) |

## Workflow

<Steps>

1. **Read the source doc.** Prefer `dialog.component.md` in the skill directory (the spec-driven source of truth for the Generation Contract, Props Interface, SCSS, and TSX). Fall back to `reference.md` if the component file is absent. Both sources yield byte-identical output (golden-guarded).

2. **Init check** ā€” run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/detect_target.py <cwd>`. If the foundation (`ui.tsx`) is missing, run Step A of `${CLAUDE_PLUGIN_ROOT}/skills/kit-core/SKILL.md` to initialize it before generating this component.

3. **Dependencies** ā€” Dialog depends on `button`. Before generating Dialog, check whether `<targetDir>/button.tsx` already exists. If missing, invoke the `component-button` skill first, then continue.

4. **Generate** ā€” apply the Generation Contract to produce:
   - `<targetDir>/dialog.tsx` from the `## Target: react` adapter in `dialog.component.md` (or the top-level `## TSX Template` section in `reference.md`)
   - `<targetDir>/dialog.module.scss` from the `## Styles` section in `dialog.component.md` (or `## SCSS Template` in `reference.md`)

5. **Verify** ā€” run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/verify_integration.py <projectRoot>` and print the summary.

</Steps>

## Accessibility

- Uses the native `<dialog>` element with `showModal()` for true modal behavior.
- `aria-modal="true"` and `aria-labelledby` wired to the dialog title.
- Focus trap implemented ā€” Tab/Shift+Tab cycles within the dialog.
- Return focus to the trigger element on close.
- Backdrop click and Escape key close the dialog.
