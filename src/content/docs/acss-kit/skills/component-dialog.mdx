The `component-dialog` skill generates an fpkit-style accessible Dialog component directly into a developer's project. It is invoked by [`/kit-add dialog`](/acss-plugins-docs/acss-kit/commands/kit-add), [`/kit-create`](/acss-plugins-docs/acss-kit/commands/kit-create) (when asking for a dialog), or by calling the `component-dialog` skill by name.

## CSS variable prefix

Dialog tokens follow the `--dlg-*` naming convention, e.g. `--dlg-bg`, `--dlg-border`, `--dlg-radius`.

## Accessibility

Dialog uses the native `<dialog>` element with `aria-modal="true"` and a focus trap that returns focus to the trigger element on close. The backdrop is dismissible unless `modal` mode is forced.

<Aside type="note">
  Dialog depends on the `component-button` skill. Running `/kit-add dialog` in a
  project that does not yet have `button.tsx` will generate Button first, then
  Dialog.
</Aside>
