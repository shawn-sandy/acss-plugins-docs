## Workflow

<Steps>

1. **Read `reference.md`** in the skill directory for canonical templates, Generation Contract, Props Interface, CSS Variables, Accessibility requirements, and Usage Examples.

2. **Init check** ‚Äî runs `detect_target.py` to verify `ui.tsx` is present. If missing, runs `kit-core` Step A initialization first.

3. **Dependencies** ‚Äî Field has no upstream component dependencies (it is a leaf component). Dependency resolution is skipped.

4. **Generate** ‚Äî writes `field.tsx` and `field.module.scss` to the configured target directory.

5. **Verify** ‚Äî runs `verify_integration.py` and prints the integration summary.

</Steps>

## Purpose

Field is a layout wrapper ‚Äî it wires together a `<label>`, an optional hint string, an optional error message, and any child input control. It generates the `id`/`htmlFor` relationship automatically and connects hint and error text via `aria-describedby`.

## Props

| Prop        | Type        | Required | Default |
| ----------- | ----------- | -------- | ------- |
| `label`     | `string`    | Yes      | ‚Äî       |
| `children`  | `ReactNode` | Yes      | ‚Äî       |
| `hint`      | `string`    | No       | ‚Äî       |
| `error`     | `string`    | No       | ‚Äî       |
| `required`  | `boolean`   | No       | `false` |
| `className` | `string`    | No       | ‚Äî       |

## CSS tokens

| Token                  | Role              | Fallback                           |
| ---------------------- | ----------------- | ---------------------------------- |
| `--field-label-size`   | Label font size   | `0.875rem`                         |
| `--field-label-weight` | Label font weight | `500`                              |
| `--field-hint-color`   | Hint text color   | `var(--color-text-muted, #6b7280)` |
| `--field-error-color`  | Error text color  | `var(--color-danger, #dc2626)`     |
| `--field-gap`          | Vertical spacing  | `0.25rem`                          |

## Accessibility

- `<label>` is always associated with the child control via `htmlFor` / `id`.
- Hint and error text are connected via `aria-describedby` on the child input.
- `aria-invalid="true"` is set on the child control when `error` is present.
- Required indicator (`*`) has `aria-hidden="true"` ‚Äî the semantic `required` attribute carries the meaning.

## Usage example

```tsx
<Field label="Email address" hint="We'll never share your email." required>
  <Input type="email" placeholder="you@example.com" />
</Field>

<Field label="Username" error="That username is already taken.">
  <Input type="text" value={username} onChange={handleChange} />
</Field>
```

<Aside type="tip">
  Field wraps any child input control ‚Äî use it with `Input`, `Checkbox`,
  `select`, or a custom control. The label association and ARIA wiring are
  handled automatically.
</Aside>
