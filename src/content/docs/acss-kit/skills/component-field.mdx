The `component-field` skill generates a self-contained accessible Field component directly into a developer's project. It is invoked via `/kit-add field` or `/kit-create` (then ask for a field).

<Aside type="note">
  This skill was added in acss-kit v1.2.0. It sets `disable-model-invocation:
  true` so it stays out of session auto-context. Invoke it explicitly via
  `/kit-add field`, `/kit-create`, or by calling `component-field` by name.
</Aside>

|                  |                                                                                                                                    |
| ---------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| **Triggered by** | `/kit-add field`, `/kit-create` (field description), direct `component-field` skill name                                           |
| **Output**       | `field.tsx` + `field.module.scss`                                                                                                  |
| **Dependencies** | None (leaf component)                                                                                                              |
| **Hint**         | Describe the label, hint text, error message, required flag, and which child input control (input/checkbox/select) the field wraps |

## Workflow

<Steps>

1. **Read the source doc.** Prefer `field.component.md` in the skill directory (the spec-driven source of truth for the Generation Contract, Props Interface, SCSS, and TSX). Fall back to `reference.md` if the component file is absent. Both sources yield byte-identical output (golden-guarded).

2. **Init check** — run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/detect_target.py <cwd>`. If the foundation (`ui.tsx`) is missing, run Step A of `${CLAUDE_PLUGIN_ROOT}/skills/kit-core/SKILL.md` to initialize it before generating this component.

3. **Dependencies** — Field has no upstream dependencies. Skip dep resolution.

4. **Generate** — apply the Generation Contract to produce:
   - `<targetDir>/field.tsx` from the `## Target: react` adapter in `field.component.md` (or the top-level `## TSX Template` section in `reference.md`)
   - `<targetDir>/field.module.scss` from the `## Styles` section in `field.component.md` (or `## SCSS Template` in `reference.md`)

5. **Verify** — run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/verify_integration.py <projectRoot>` and print the summary.

</Steps>

## Accessibility

- `<label>` is always rendered and associated with the child input via `htmlFor`/`id` pairing.
- Hint text and error message are associated via `aria-describedby`.
- `aria-required` and `aria-invalid` propagate to the child control when `required` or `error` props are set.
- Error message uses `role="alert"` so screen readers announce it immediately when it appears.
