The `component-field` skill generates an fpkit-style accessible Field component directly into a developer's project. It is invoked by [`/kit-add field`](/acss-plugins-docs/acss-kit/commands/kit-add), [`/kit-create`](/acss-plugins-docs/acss-kit/commands/kit-create) (when asking for a field), or by calling the `component-field` skill by name.

|                  |                                                                                               |
| ---------------- | --------------------------------------------------------------------------------------------- |
| **Triggered by** | `/kit-add field`, `/kit-create` ‚Üí "field", or direct skill invocation                         |
| **Output**       | `<targetDir>/field.tsx` + `<targetDir>/field.module.scss`                                     |
| **Dependencies** | None ‚Äî leaf component                                                                         |
| **Hint**         | Describe whether you need hint text, an error message slot, and whether the field is required |

## Workflow

<Steps>

1. **Read** `reference.md` in the `component-field` skill directory for the canonical TSX/SCSS templates, Generation Contract, Props Interface, CSS Variables, Accessibility requirements, and Usage Examples.

2. **Init check** ‚Äî run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/detect_target.py <cwd>`. If the foundation (`ui.tsx`) is missing, run Step A of the [`kit-core` skill](/acss-plugins-docs/acss-kit/skills/kit-core) to initialize it before generating this component.

3. **Dependencies** ‚Äî Field has no upstream dependencies. Skip dep resolution.

4. **Generate** ‚Äî apply the Generation Contract from `reference.md` to produce:
   - `<targetDir>/field.tsx` from the `## TSX Template` section
   - `<targetDir>/field.module.scss` from the `## SCSS Template` section

5. **Verify** ‚Äî run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/verify_integration.py <projectRoot>` and print the summary to the developer.

</Steps>

## Invocation hints

```text
/kit-add field
```

```text
/kit-create ‚Äî "create a field wrapper with hint text and an error message slot"
```

## CSS variable prefix

Field tokens follow the `--field-*` naming convention, e.g. `--field-label-color`, `--field-hint-color`, `--field-error-color`.

## Accessibility

Field associates its label, hint, and error message with the wrapped input via `aria-describedby`. Required fields use `aria-required="true"` on the inner control.
