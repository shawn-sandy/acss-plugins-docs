The `component-popover` skill generates a self-contained accessible Popover component directly into a developer's project. It is invoked via `/kit-add popover` or `/kit-create` (then ask for a popover).

<Aside type="note">
  This skill was added in acss-kit v1.2.0. It sets `disable-model-invocation:
  true` so it stays out of session auto-context. Invoke it explicitly via
  `/kit-add popover`, `/kit-create`, or by calling `component-popover` by name.
</Aside>

## Accessibility

- Uses the [Popover API](https://developer.mozilla.org/en-US/docs/Web/API/Popover_API) (`popover` attribute) for native browser light-dismiss and top-layer stacking.
- `aria-expanded` on the trigger button reflects the open/closed state.
- Focus trap: when `focusTrap` prop is set, Tab/Shift+Tab cycles within the popover content.
- `aria-haspopup` on the trigger describes the control type to assistive technologies.
