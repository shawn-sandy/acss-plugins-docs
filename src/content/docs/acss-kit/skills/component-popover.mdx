The `component-popover` skill generates a self-contained accessible Popover component directly into a developer's project. It is invoked via `/kit-add popover` or `/kit-create` (then ask for a popover).

<Aside type="note">
  This skill was added in acss-kit v1.2.0. It sets `disable-model-invocation:
  true` so it stays out of session auto-context. Invoke it explicitly via
  `/kit-add popover`, `/kit-create`, or by calling `component-popover` by name.
</Aside>

|                  |                                                                                                                                        |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| **Triggered by** | `/kit-add popover`, `/kit-create` (popover description), direct `component-popover` skill name                                         |
| **Output**       | `popover.tsx` + `popover.module.scss`                                                                                                  |
| **Dependencies** | None (leaf component)                                                                                                                  |
| **Hint**         | Describe the trigger element, the popover content, anchor/positioning, and whether light-dismiss and focus-trap behaviour are required |

## Workflow

<Steps>

1. **Read the source doc.** Prefer `popover.component.md` in the skill directory (the spec-driven source of truth for the Generation Contract, Props Interface, SCSS, and TSX). Fall back to `reference.md` if the component file is absent. Both sources yield byte-identical output (golden-guarded).

2. **Init check** — run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/detect_target.py <cwd>`. If the foundation (`ui.tsx`) is missing, run Step A of `${CLAUDE_PLUGIN_ROOT}/skills/kit-core/SKILL.md` to initialize it before generating this component.

3. **Dependencies** — Popover has no upstream dependencies. Skip dep resolution.

4. **Generate** — apply the Generation Contract to produce:
   - `<targetDir>/popover.tsx` from the `## Target: react` adapter in `popover.component.md` (or the top-level `## TSX Template` section in `reference.md`)
   - `<targetDir>/popover.module.scss` from the `## Styles` section in `popover.component.md` (or `## SCSS Template` in `reference.md`)

5. **Verify** — run `python3 ${CLAUDE_PLUGIN_ROOT}/scripts/verify_integration.py <projectRoot>` and print the summary.

</Steps>

## Accessibility

- Uses the [Popover API](https://developer.mozilla.org/en-US/docs/Web/API/Popover_API) (`popover` attribute) for native browser light-dismiss and top-layer stacking.
- `aria-expanded` on the trigger button reflects the open/closed state.
- Focus trap: when `focusTrap` prop is set, Tab/Shift+Tab cycles within the popover content.
- `aria-haspopup` on the trigger describes the control type to assistive technologies.
