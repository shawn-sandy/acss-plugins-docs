The `component-input` skill generates an fpkit-style accessible Input component directly into a developer's project. It is invoked by [`/kit-add input`](/acss-plugins-docs/acss-kit/commands/kit-add) or [`/kit-create`](/acss-plugins-docs/acss-kit/commands/kit-create).

## Workflow

<Steps>

1. **Read `reference.md`** in the skill directory for canonical templates, Generation Contract, Props Interface, CSS Variables, Accessibility requirements, and Usage Examples.

2. **Init check** тАФ runs `detect_target.py` to verify `ui.tsx` is present. If missing, runs `kit-core` Step A initialization first.

3. **Dependencies** тАФ Input has no upstream component dependencies (it is a leaf component). Dependency resolution is skipped.

4. **Generate** тАФ writes `input.tsx` and `input.module.scss` to the configured target directory.

5. **Verify** тАФ runs `verify_integration.py` and prints the integration summary.

</Steps>

## Accessibility

- `aria-invalid="true"` is set when `error={true}`.
- `aria-required` is set when `required={true}`.
- `aria-disabled="true"` pattern is used instead of the native `disabled` attribute to preserve focusability.
- Focus ring uses `--color-focus-ring` for WCAG-compliant visibility.

## Controlled vs uncontrolled

Pass `value` + `onChange` for a controlled input, or `defaultValue` alone for an uncontrolled input. Both patterns are supported.

```tsx
{
  /* Controlled */
}
<Input type="email" value={email} onChange={(e) => setEmail(e.target.value)} />;

{
  /* Uncontrolled */
}
<Input type="text" defaultValue="Initial value" />;
```
