The `component-card` skill generates an fpkit-style accessible Card component directly into a developer's project. It is invoked by [`/kit-add card`](/acss-plugins-docs/acss-kit/commands/kit-add) or [`/kit-create`](/acss-plugins-docs/acss-kit/commands/kit-create).

## Workflow

<Steps>

1. **Read `reference.md`** in the skill directory for canonical templates, Generation Contract, Props Interface, CSS Variables, Accessibility requirements, and Usage Examples.

2. **Init check** ‚Äî runs `detect_target.py` to verify `ui.tsx` is present. If missing, runs `kit-core` Step A initialization first.

3. **Dependencies** ‚Äî Card has no upstream component dependencies (it is a leaf component). Dependency resolution is skipped.

4. **Generate** ‚Äî writes `card.tsx` and `card.module.scss` to the configured target directory.

5. **Verify** ‚Äî runs `verify_integration.py` and prints the integration summary.

</Steps>

## Slots

Card exposes three named slots:

| Slot   | Prop       | Description                          |
| ------ | ---------- | ------------------------------------ |
| Header | `header`   | Card title or leading content area   |
| Body   | `children` | Main content (always rendered)       |
| Footer | `footer`   | Action row or metadata at the bottom |

## Props

| Prop          | Type        | Required | Default |
| ------------- | ----------- | -------- | ------- |
| `children`    | `ReactNode` | Yes      | ‚Äî       |
| `header`      | `ReactNode` | No       | ‚Äî       |
| `footer`      | `ReactNode` | No       | ‚Äî       |
| `interactive` | `boolean`   | No       | `false` |
| `href`        | `string`    | No       | ‚Äî       |
| `className`   | `string`    | No       | ‚Äî       |

## CSS tokens

| Token              | Role               | Fallback                       |
| ------------------ | ------------------ | ------------------------------ |
| `--card-bg`        | Background color   | `var(--color-surface, #fff)`   |
| `--card-border`    | Border color       | `var(--color-border, #e5e7eb)` |
| `--card-radius`    | Border radius      | `var(--radius-lg, 0.5rem)`     |
| `--card-padding-x` | Horizontal padding | `1.25rem`                      |
| `--card-padding-y` | Vertical padding   | `1.25rem`                      |
| `--card-shadow`    | Box shadow         | `none`                         |

## Accessibility

- Interactive cards (`interactive={true}` or `href` set) use a `<button>` or `<a>` wrapper with keyboard activation.
- `card:activate` custom event is dispatched on activation for vanilla-JS consumers.
- Non-interactive cards are plain `<div>` with no redundant ARIA.

## Usage example

```tsx
<Card
  header={<h2>Pricing plan</h2>}
  footer={<Button variant="primary">Get started</Button>}
>
  Premium tier with all features unlocked.
</Card>
```

<Aside type="tip">
  Pass `interactive` and an `href` to make the entire card keyboard-navigable as
  a link ‚Äî useful for content grids and dashboards.
</Aside>
