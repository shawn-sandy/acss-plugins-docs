<Aside type="danger">
  `/kit-add-html` was **removed in acss-kit v1.1.0**. Use `/kit-add
  --target=html <name>` instead. The alias no longer exists in the plugin.
</Aside>

`/kit-add-html` previously generated static HTML + SCSS + vanilla JS component
files for non-React projects. That workflow is now the **HTML Target** section
of the [`kit-core` skill](/acss-plugins-docs/acss-kit/skills/kit-core) and is
accessed via `/kit-add --target=html`.

## Migration

| Old command                              | New command                                       |
| ---------------------------------------- | ------------------------------------------------- |
| `/kit-add-html button`                   | `/kit-add --target=html button`                   |
| `/kit-add-html card alert`               | `/kit-add --target=html card alert`               |
| `/kit-add-html button card alert dialog` | `/kit-add --target=html button card alert dialog` |

## What happened

In acss-kit v0.13.0, the React and HTML generation workflows were unified into
the single `components` skill. The HTML Target workflow lives under the
`## HTML Target` section (steps HT-A through HT-F) of that skill.

In acss-kit v1.1.0, the `/kit-add-html` alias was fully removed. The `components`
skill was also renamed to `kit-core` in v1.2.0. All HTML-target detection runs
through `detect_target.py` used for React output.

<Aside type="tip">
  See the [`kit-core` skill](/acss-plugins-docs/acss-kit/skills/kit-core) for
  the full orchestration workflow, including Creator Mode, Form Mode, and
  HTML/Style-Tune modes.
</Aside>
