<Aside type="danger">
  `/kit-add-html` has been **removed from the plugin** as of acss-kit v1.1.0
  (2026-05-21). The command file (`plugins/acss-kit/commands/kit-add-html.md`)
  was deleted from the upstream repository. Use
  [`/kit-add`](/acss-plugins-docs/acss-kit/commands/kit-add) for all component
  generation.
</Aside>

`/kit-add-html` was a deprecated alias that forwarded to `/kit-add
--target=html`. The `--target=html` flag itself was retired when the unified
per-component skill architecture landed in v1.2.0 — every component now has its
own `component-<name>/SKILL.md` that handles both React and HTML output directly.

## Migration

| Old command                              | Replacement                         |
| ---------------------------------------- | ----------------------------------- |
| `/kit-add-html button`                   | `/kit-add button`                   |
| `/kit-add-html card alert`               | `/kit-add card alert`               |
| `/kit-add-html button card alert dialog` | `/kit-add button card alert dialog` |

## What happened

- **v0.13.0** — React and HTML generation unified into the `components` skill; `/kit-add-html` became a forwarding alias.
- **v1.0.0** — Alias retained but marked deprecated; printed a deprecation notice at runtime.
- **v1.1.0** — Alias command file deleted from upstream. Use `/kit-add` directly.

<Aside type="tip">
  See [`/kit-add`](/acss-plugins-docs/acss-kit/commands/kit-add) for the current
  component generation workflow, including the full component catalogue,
  dependency resolution, and per-component skills.
</Aside>
