<Aside type="danger">
  `/kit-add-html` has been **removed from upstream** as of acss-kit v1.1.0. The
  command file (`plugins/acss-kit/commands/kit-add-html.md`) was deleted. Use
  [`/kit-add
  --target=html`](/acss-plugins-docs/acss-kit/commands/kit-add#html-target) for
  static HTML output.
</Aside>

`/kit-add-html` was a thin alias that forwarded to `/kit-add --target=html`. That
alias was removed in v1.1.0. The underlying HTML generation workflow (steps HT-A
through HT-F) still exists inside [`kit-core`](/acss-plugins-docs/acss-kit/skills/kit-core);
invoke it directly via `/kit-add --target=html`.

## Migration

| Old command                              | Replacement                                       |
| ---------------------------------------- | ------------------------------------------------- |
| `/kit-add-html button`                   | `/kit-add --target=html button`                   |
| `/kit-add-html card alert`               | `/kit-add --target=html card alert`               |
| `/kit-add-html button card alert dialog` | `/kit-add --target=html button card alert dialog` |

## What happened

As of acss-kit v0.13.0, the React and HTML generation workflows were unified into
the single `components` skill (later renamed `kit-core`). The `kit-add-html.md`
command alias was subsequently removed in v1.1.0 â€” it had been a thin wrapper
around `/kit-add --target=html` since v0.13.0.

The `components-html` skill file and the `detect_html_target.py` script were
also removed upstream. All component generation now routes through dedicated
per-component skills at `skills/component-<name>/SKILL.md`.

<Aside type="tip">
  See [`/kit-add
  --target=html`](/acss-plugins-docs/acss-kit/commands/kit-add#html-target) for
  the static HTML generation workflow and the list of HTML-capable components,
  and [`kit-core skill`](/acss-plugins-docs/acss-kit/skills/kit-core) for
  orchestration details.
</Aside>
