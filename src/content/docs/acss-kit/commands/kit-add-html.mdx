<Aside type="danger">
  `/kit-add-html` has been **removed from upstream** as of acss-kit v1.2.x. The
  command file (`plugins/acss-kit/commands/kit-add-html.md`) was deleted. Use
  [`/kit-add`](/acss-plugins-docs/acss-kit/commands/kit-add) for all component
  generation going forward.
</Aside>

`/kit-add-html` previously generated static HTML + SCSS + vanilla JS component
files for non-React projects. That workflow was previously the **HTML Target**
section of the `components` skill (now [`kit-core`](/acss-plugins-docs/acss-kit/skills/kit-core)).

## Migration

| Old command                              | Replacement                         |
| ---------------------------------------- | ----------------------------------- |
| `/kit-add-html button`                   | `/kit-add button`                   |
| `/kit-add-html card alert`               | `/kit-add card alert`               |
| `/kit-add-html button card alert dialog` | `/kit-add button card alert dialog` |

## What happened

As of acss-kit v0.13.0, the React and HTML generation workflows were unified into
the single `components` skill (later renamed `kit-core`). The `kit-add-html.md`
command file was subsequently removed upstream in v1.2.x.

The `components-html` skill file and the `detect_html_target.py` script were
also removed upstream. All component generation now routes through dedicated
per-component skills at `skills/component-<name>/SKILL.md`.

<Aside type="tip">
  See [`/kit-add`](/acss-plugins-docs/acss-kit/commands/kit-add) for the current
  component generation workflow and [`kit-core
  skill`](/acss-plugins-docs/acss-kit/skills/kit-core) for orchestration
  details.
</Aside>
