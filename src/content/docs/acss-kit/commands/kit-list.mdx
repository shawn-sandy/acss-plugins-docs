Browse the component catalogue or inspect a specific component's props, CSS tokens, dependencies, and generation contract. This command is entirely read-only — it never writes files.

## Syntax

```text
/kit-list [component]
```

## Arguments

| Argument    | Required | Description                                                      |
| ----------- | -------- | ---------------------------------------------------------------- |
| `component` | No       | Component name to inspect. If omitted, shows the full catalogue. |

## Examples

List all available components:

```text
/kit-list
```

Inspect a specific component:

```text
/kit-list dialog
/kit-list button
/kit-list card
```

## Output: full catalogue

Without a component name, `/kit-list` reads the `name:` and `description:` front-matter from every `skills/component-*/SKILL.md` file and prints a categorized listing:

```

  img         — Accessible image with alt, lazy-loading, aspect-ratio
  link        — Anchor with external detection and current-page marking
  list        — Ordered/unordered list with role="list" reset

Interactive:
  button      — Primary interactive element (all variants)
  checkbox    — Checkbox with indeterminate state and aria-checked
  icon-button — Icon-only button with required aria-label

Form:
  field       — Form field wrapper with label, hint, and error message
  input       — Text/email/number input with aria-invalid and error state

Layout:
  alert       — Severity-aware notification with ARIA live regions
  card        — Content container with header/body/footer slots

Complex (with dependencies):
  dialog      — Modal dialog with focus trap (needs button)

  table       — Data table with caption, scope headers, and sort support

Run /kit-add <component> to generate any component.
```

## Reading the generation contract

The generation contract tells you exactly what `/kit-add <component>` will create, including the resolved dependency tree. This lets you preview the output before running the write command.

<Aside type="tip">
Use `/kit-list <component>` as a planning step before `/kit-add`. You can see the full dependency chain and all CSS tokens without touching your project files.
</Aside>

## No side effects

`/kit-list` never creates, modifies, or deletes any file. It's safe to run at any time, in any project state.
