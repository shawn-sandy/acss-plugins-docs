`/style-tune` is the explicit slash-command form of the `style-tune` skill. The skill also auto-triggers on natural-language phrases ("warmer button", "more elevated cards"), so you rarely need to type the slash command ‚ÄĒ but it's available when you want to be explicit about scope.

## Syntax

```text
/style-tune <subject> <modifier(s)>
```

## Examples

```text
/style-tune button softer warmer
/style-tune primary "tone down"
/style-tune card "more elevated"
/style-tune input sharper
```

Or, equivalently, just write the intent in chat:

> Make the button feel softer and warmer.
> Tone down the primary color a touch.
> More elevated cards.

## Scope

The skill edits one of two layers:

| Layer | Targets |
|---|---|
| Theme roles | `primary`, `accent`, `danger`, `warning`, `info`, `success`, `brand` |
| Component tokens | `button`, `card`, `alert`, `dialog`, `input`, `nav` |

## Modifier vocabulary

These match the canonical mappings in the [`style-tune` skill](/acss-plugins-docs/acss-kit/skills/style-tune#intent-vocabulary):

| Modifier | Target tokens | Effect |
|---|---|---|
| warmer | `--*-bg`, `--color-*` | Hue shift toward amber (‚Čą +15‚Äď30¬į) |
| cooler | `--*-bg`, `--color-*` | Hue shift toward blue (‚Čą ‚ąí15‚Äď30¬į) |
| softer | `--*-radius` | Radius √ó 1.5 (min 4px) |
| sharper | `--*-radius` | Radius √ó 0.5 (min 0) |
| larger / bigger | `--*-font-size`, `--*-padding-*` | √ó 1.125 |
| smaller | `--*-font-size`, `--*-padding-*` | √ó 0.875 |
| bolder | `--*-bg` / `--color-*` chroma | Chroma √ó 1.2 |
| subtler / muted / tone down | `--*-bg` / `--color-*` chroma | Chroma √ó 0.8 |
| more elevated | `--*-shadow` | Next shadow step up |
| flatter | `--*-shadow` | Next shadow step down (or removed) |

## Workflow

1. Identify the subject and modifiers, then route to the theme or component layer.
2. Locate the relevant theme files or component SCSS.
3. Compute adjustments ‚ÄĒ colours via `oklch_shift.py`, spatial tokens via canonical multipliers.
4. Apply edits with validation:
   - Theme changes are pre-validated for WCAG contrast in a temp directory; failures roll back the entire batch.
   - Component edits also run a WCAG pre-check on the affected token against its typical background role, and preserve `var()` wrappers and fallbacks.
5. Re-validate the file structure and surface drift hints.
6. Print a summary table ‚ÄĒ modifier, token, old value, new value, status.

<Aside type="tip">
Because `/style-tune` always edits CSS custom properties (never raw colour literals), you can override or revert any single token by editing your own stylesheet without touching the generated file.
</Aside>
