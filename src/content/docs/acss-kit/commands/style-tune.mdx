`/style-tune` is the explicit slash-command form of the `style-tune` skill. The skill also auto-triggers on natural-language phrases ("warmer button", "calmer alert"), so you rarely need to type the slash command â€” but it's available when you want to be explicit about scope.

## Syntax

```text
/style-tune <subject> <modifier(s)>
```

## Examples

```text
/style-tune button softer warmer
/style-tune primary tone-down
/style-tune alert calmer
/style-tune card more-spacious
```

Or, equivalently, just write the intent in chat:

> Make the button feel softer and warmer.
> Tone down the primary color a touch.
> More spacious cards.

## Scope

The skill edits one of two layers:

| Layer | Targets |
|---|---|
| Theme roles | `primary`, `accent`, `danger`, `warning`, `info`, `success`, `brand` |
| Component tokens | `button`, `card`, `alert`, `dialog`, `input`, `nav` |

## Modifier vocabulary

| Modifier | Meaning |
|---|---|
| softer | Lower chroma, gentler radius |
| warmer | Hue shift toward red/orange |
| cooler | Hue shift toward blue/teal |
| calmer | Lower chroma, lower contrast accent |
| tone-down | Reduce saturation while preserving role |
| more-spacious | Larger paddings/gaps |
| elevated | Stronger shadow + slight radius bump |

## Workflow

1. Identify the subject and modifiers, then route to the theme or component layer.
2. Locate the relevant theme files or component SCSS.
3. Compute adjustments â€” colours via `oklch_shift.py`, spatial tokens via canonical multipliers.
4. Apply edits with validation:
   - Theme changes are pre-validated for WCAG contrast in a temp directory; failures roll back the entire batch.
   - Component edits preserve `var()` wrappers and fallbacks.
5. Re-validate the file structure and surface drift hints.
6. Print a summary table â€” modifier, token, old value, new value, status.

<Aside type="tip">
Because `/style-tune` always edits CSS custom properties (never raw colour literals), you can override or revert any single token by editing your own stylesheet without touching the generated file.
</Aside>
